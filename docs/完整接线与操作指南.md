## 二、完整接线表

### 2.1 RA6M5 → PCA9685（I2C 舵机控制）

| RA6M5 引脚 | PCA9685 引脚 | 线色建议 | 说明 |
|-----------|-------------|---------|------|
| **P511** | SDA | 蓝色 | I2C 数据线 |
| **P512** | SCL | 黄色 | I2C 时钟线 |
| **3V3** | VCC | 红色 | 逻辑供电（必须 3.3V，不能 5V） |
| **GND** | GND | 黑色 | 共地 |

PCA9685 的 **OE 引脚**悬空或接 GND（接 GND = 使能输出）。

### 2.2 PCA9685 → 6 路舵机

| PCA9685 通道 | 关节 | 舵机规格 | 舵机线颜色 |
|-------------|------|---------|-----------|
| **CH0** | 肘关节 (Elbow) | 20kg / 180° | 棕=GND, 红=V+, 橙=信号 |
| **CH1** | 肩关节 (Shoulder) | 25kg / 180° | 同上 |
| **CH2** | 腕部俯仰 (Wrist Pitch) | 20kg / 180° | 同上 |
| **CH3** | 腕部旋转 (Wrist Rotate) | 20kg / 180° | 同上 |
| **CH4** | 夹爪开合 (Gripper) | 20kg / 180° | 同上 |
| **CH5** | 底座旋转 (Base) | 30kg / 270° | 同上 |

PCA9685 舵机电源端（绿色接线柱）：

| 接线柱 | 接到 | 说明 |
|--------|------|------|
| **V+** | DC-DC 降压模块输出正极 | **6V**（不是 12V！） |
| **GND** | DC-DC 降压模块输出负极 | 与 MCU GND 共地 |

### 2.3 RA6M5 → 树莓派 5（UART 串口通信）

| RA6M5 引脚 | Pi 5 GPIO | Pi 5 物理针脚 | 线色建议 | 说明 |
|-----------|----------|-------------|---------|------|
| **P109** (TXD9) | GPIO15 (RXD) | **Pin 10** | 橙色 | MCU 发 → Pi 收 |
| **P110** (RXD9) | GPIO14 (TXD) | **Pin 8** | 绿色 | Pi 发 → MCU 收 |
| **GND** | GND | **Pin 6** 或 **Pin 9** | 黑色 | 共地（必须接！） |

> **注意：TX 接对方 RX，是交叉连接，不是同名对接！**
> 两端都是 3.3V 电平，直连即可，不需要电平转换。

### 2.4 RA6M5 → 继电器驱动模块 → 传送带

| RA6M5 引脚 | 驱动模块引脚 | 说明 |
|-----------|------------|------|
| **P415** | IN (信号输入) | GPIO 高电平 = 继电器吸合 = 传送带转 |
| **GND** | GND | 共地 |
| — | VCC | 接 **5V**（从 MCU 板 5V0 排针取） |

继电器输出端：

| 继电器端子 | 接到 | 说明 |
|----------|------|------|
| **COM** | 传送带电机电源正极 | 公共端 |
| **NO** | 传送带电机 | 常开端（通电时闭合） |

### 2.5 RA6M5 → FSR402B 压力传感器（新增）

**两个传感器模块，分别贴在夹爪的左右指上。**

| 传感器 | 模块引脚 | RA6M5 引脚 | 说明 |
|-------|---------|-----------|------|
| **FSR-左指** | VCC | **3V3** | ⚠️ 必须 3.3V，接 5V 会烧 ADC |
| | GND | **GND** | 共地 |
| | **AO** | **P001** (AN001) | 模拟电压输出 → ADC0 通道 1 |
| | DO | 不接 | 数字输出，本项目不用 |
| | LED+ | 不接 | 指示灯，悬空即可 |
| **FSR-右指** | VCC | **3V3** | 同上 |
| | GND | **GND** | 共地 |
| | **AO** | **P003** (AN003) | 模拟电压输出 → ADC0 通道 3 |
| | DO | 不接 | 同上 |
| | LED+ | 不接 | 同上 |

**FSR 模块上的蓝色电位器**：调节 DO 输出的阈值，我们不用 DO，所以不需要调。

### 2.6 P001 和 P003 在板上的位置

从 V4.0 板子**背面**看，底部排针标注（从左到右）：

```
GND GND 5V0 3V3  405 001  415 414  203 204  202 205  113 112 ...
                      ↑        ↑
                   P001      P415
                  (FSR左)   (继电器)
```

P003 在底部排针靠右侧的位置（`503 003 106` 附近）。

## 三、供电方案

```
┌─────────────────────────────────────────────────────────┐
│ 电源 1：USB (电脑/充电头)                                  │
│   → RA6M5 开发板 (5V)                                    │
│       ├── 板载 3.3V 稳压 → PCA9685 逻辑 VCC              │
│       ├── 板载 3.3V 稳压 → FSR402B 模块 ×2 VCC           │
│       ├── 板载 5V0 排针 → 继电器驱动模块 VCC              │
│       └── GND ──────────────────┐                       │
│                                  │                       │
│ 电源 2：外部 12V 电源              │ (共地)               │
│   → DC-DC 降压模块 (输出 6V/5A)   │                       │
│       ├── V+ → PCA9685 舵机电源正  │                       │
│       └── GND ──────────────────┤                       │
│                                  │                       │
│ 电源 3：USB-C (5V/3A 以上)        │                       │
│   → 树莓派 5                      │                       │
│       └── GND ──────────────────┘                       │
└─────────────────────────────────────────────────────────┘
```

**三路电源完全独立，但所有 GND 必须汇合成一个共地网络。**

## 五、安全检查清单

### 5.1 接线前（断电状态）

- [ ] 所有杜邦线公头无裸露短路风险
- [ ] PCA9685 逻辑 VCC 接的是 **3.3V**（不是 5V）
- [ ] FSR 模块 VCC 接的是 **3.3V**（不是 5V）
- [ ] 继电器驱动模块 VCC 接的是 **5V**（不是 3.3V）
- [ ] 舵机电源接的是降压后的 **6V**（不是 12V 直接输入）
- [ ] Pi TX (Pin8) → MCU P110 (RX)，Pi RX (Pin10) → MCU P109 (TX)，**交叉连接**
- [ ] 3.3V 线没有碰到 GND（万用表蜂鸣档确认不短路）
- [ ] 5V 线没有碰到 3.3V 线

### 5.2 共地验证（万用表蜂鸣档/电阻档）

以下任意两点之间阻值应 **< 1Ω**：

- [ ] MCU GND ↔ PCA9685 逻辑 GND
- [ ] MCU GND ↔ PCA9685 舵机电源 GND
- [ ] MCU GND ↔ Pi GND (Pin 6)
- [ ] MCU GND ↔ 继电器驱动模块 GND
- [ ] MCU GND ↔ FSR-左指模块 GND
- [ ] MCU GND ↔ FSR-右指模块 GND

### 5.3 FSR 模块上电前单独测试

**在接到 MCU 之前，先单独测试模块输出电压：**

```
步骤：
1. FSR 模块 VCC 接 MCU 板 3.3V，GND 接 MCU 板 GND
2. 万用表红表笔接模块 AO 引脚，黑表笔接 GND
3. 不按压传感器 → 读数应接近 0V（或 3.3V，取决于模块设计）
4. 手指按压传感器 → 读数应在 0V ~ 3.3V 之间平滑变化
5. 用力按压 → 确认最大电压 ≤ 3.3V

判定：
  ✅ 最大电压 ≤ 3.3V → 安全，可以接 MCU P001/P003
  ❌ 最大电压 > 3.3V → 停止！不能直接接 MCU，需要加分压电阻
```

**如果需要加分压：**

```
模块 AO ──── 10KΩ ──┬── MCU P001/P003
                     │
                   10KΩ
                     │
                    GND

输出电压 = AO电压 × 0.5，确保不超过 3.3V
```

## 六、上电/断电操作流程

### 6.1 上电顺序（严格按此顺序）

```
第 1 步：目视检查所有接线，过一遍第五章检查清单
           ↓
第 2 步：接通 12V 外部电源 → DC-DC 降压 → PCA9685 舵机电源
         （用万用表确认降压输出是 5.5~6.5V，不是 12V）
           ↓
第 3 步：USB 接 MCU（插电脑或充电头）
         → 板载 PWR LED 亮起
         → 舵机应自动回到 HOME 姿态
         → 调试串口输出 [BOOT] 信息
           ↓
第 4 步：USB-C 接 Pi 5
         → Pi 启动（红灯常亮，绿灯闪烁）
         → SSH 连接 Pi
           ↓
第 5 步：在 Pi 上启动程序
         $ cd ~/vs_code/strawberry_grasp
         $ source ~/vs_code/.venv/bin/activate
         $ python main.py --dry-run --preview
           ↓
第 6 步：PC 浏览器打开 http://<Pi_IP>:8080 查看实时画面
```

### 6.2 断电顺序（反过来）

```
第 1 步：Pi 上 Ctrl+C 停止程序
           ↓
第 2 步：Pi 关机
         $ sudo shutdown -h now
         （等绿灯熄灭后再断电）
           ↓
第 3 步：拔 Pi USB-C 电源
           ↓
第 4 步：拔 MCU USB
           ↓
第 5 步：关闭 12V 外部电源
```

### 6.3 紧急停止

任何异常情况（舵机异响、冒烟、发热）：

```
立即操作（按优先级）：
1. 拔 12V 外部电源（断舵机动力）
2. 拔 MCU USB（断控制信号）
3. 拔 Pi USB-C
4. 检查接线是否有短路/接反
```

软件紧急停止：
- 调试串口发送 `x` 或 `X`
- Pi 端程序会发送 `X` 命令
- MCU 收到后：停传送带 → 回 HOME → 进入 IDLE

## 七、全链路联调流程

### 7.1 分步验证（先单独测，再联合）

| 步骤 | 测试内容 | 操作 | 预期结果 |
|------|---------|------|---------|
| 1 | MCU 基础启动 | USB 接 MCU，打开串口终端(115200) | 看到 `[BOOT]` 日志 |
| 2 | 舵机 HOME | 上舵机电源 | 机械臂自动回 HOME 姿态 |
| 3 | 示教模式 | 串口发送 `t` | 进入示教，W/S 调关节角度 |
| 4 | 传送带控制 | 串口发送 `g`（启动），`x`（停止） | 传送带正确启停 |
| 5 | Pi 视觉 dry-run | Pi 运行 `python main.py --dry-run --preview` | 浏览器看到实时画面+检测框 |
| 6 | Pi→MCU 串口 | Pi 运行 `python main.py --preview`（不加 dry-run） | MCU 串口输出 `[PI] Classification: A/B/C` |
| 7 | 全链路闭环 | Pi 发送 G 启动传送带，放草莓，等待检测 | 检测→停带→抓取→分拣→放置→回HOME |

### 7.2 全链路运行命令

**Pi 端：**
```bash
cd ~/vs_code/strawberry_grasp
source ~/vs_code/.venv/bin/activate
python main.py --preview
```

**PC 端：**
- 浏览器打开 `http://<Pi_IP>:8080` 查看实时画面
- 串口终端连接 MCU（通过板载 DAP），波特率 115200，查看状态机日志

### 7.3 完整自动分拣流程

```
MCU 上电 → HOME 姿态 → IDLE 等待
                ↓
Pi 发送 'G' → MCU 进入 BELT_RUN → 继电器闭合 → 传送带启动
                ↓
草莓经过摄像头 → YOLO 检测 → Pi 发送分类指令 (A/B/C)
                ↓
MCU 收到指令 → BELT_STOP → 传送带停止 → 等 500ms
                ↓
PRE_GRASP → 手臂接近草莓
                ↓
GRASP → 夹爪闭合抓取 → 等 800ms
                ↓
LIFT → 抬起草莓 → 等 800ms
                ↓
PLACE → 抬高到 TRANSIT 安全高度 → 旋转底座到目标位置 → 降下放置
        A(ripe) → 176°方向
        B(semi_ripe) → 201°方向
        C(unripe) → 223°方向
                ↓
RETURN → 抬高 → 旋转回正 → 降到 HOME → 发送 READY 给 Pi
                ↓
Pi 收到 READY → 发送 'G' → 新一轮循环
```

## 八、压力传感器集成步骤（后续）

### 8.1 硬件准备

1. 将 FSR402B 薄膜传感器贴到夹爪左右指内侧
2. 传感器引线连接到转换模块的两针接口（棕=传感器线，红=传感器线）
3. 转换模块 4 针端按上表接到 MCU

### 8.2 软件集成路线

```
第 1 步：e2studio FSP 配置
  - 添加 ADC0 Stack
  - 配置通道 AN001 + AN003
  - 配置扫描模式：单次扫描 (One-Shot)
  - 分辨率：12-bit
  - 参考电压：内部 3.3V

第 2 步：hal_entry.c 中添加 ADC 代码
  - 初始化 ADC
  - 在 GRASP 状态中采样压力值
  - 当压力超过阈值 → 停止夹紧（防止压伤草莓）
  - 当压力低于阈值 → 检测到滑脱 → 略微加紧

第 3 步：标定
  - 空载时的 ADC 读数 → 基线值
  - 轻触草莓时的读数 → 接触阈值
  - 正常夹持时的读数 → 目标夹持力
  - 草莓变形时的读数 → 过压阈值（必须低于此值）

第 4 步：集成到状态机
  STATE_GRASP 中：
    闭合夹爪 → 每 5ms 读一次 ADC
    → 压力达到目标值 → 停止闭合 → 进入 LIFT
    → 压力超过过压阈值 → 微松夹爪 → 报警
    → 超时未达到目标压力 → 抓取失败 → 回 HOME
```

### 8.3 安全阈值建议

| 状态 | ADC 读数范围 (12-bit, 0~4095) | 含义 |
|------|------------------------------|------|
| 空载 | 0 ~ 100 | 未接触 |
| 轻触 | 100 ~ 500 | 刚接触草莓 |
| 正常夹持 | 500 ~ 1500 | 目标力度 |
| 过压警告 | 1500 ~ 2000 | 力度偏大，微松 |
| 过压保护 | > 2000 | 立即松开，防止压伤 |

> 以上为参考值，实际阈值需要用真实草莓标定后确定。

## 九、故障排查

| 现象 | 可能原因 | 排查方法 |
|------|---------|---------|
| MCU 上电后无 [BOOT] 日志 | 串口未连接 / 波特率错误 | 确认串口终端 115200-8N1 |
| 舵机不动 | 舵机电源未接 / I2C 线松了 | 检查 PCA9685 供电和 SDA/SCL |
| 舵机抖动 | 供电不足 / 降压模块过载 | 用万用表量舵机电源端电压是否稳定 |
| Pi 和 MCU 无法通信 | TX/RX 接反 / GND 没共地 | 交换 TX/RX 线；确认 GND 连通 |
| 传送带不转 | P415 线松 / 继电器模块供电不足 | 检查 P415 → 驱动模块 IN 连接 |
| 传送带一直转不停 | 继电器常闭端接错 | 应该接 COM + NO（常开） |
| ADC 读数始终为 0 | FSR 模块未供电 / AO 线断了 | 万用表量 AO 引脚有无电压变化 |
| ADC 读数始终为 4095 | 模块 VCC 接了 5V / AO 电压超 3.3V | 断电！换接 3.3V |
| 机械臂碰撞 | 姿态角度不对 / 舵机 reversed 错误 | 用示教模式(T)逐个关节调试 |
