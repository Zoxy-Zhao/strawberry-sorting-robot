# RA6M5 下位机实施清单

> 更新于 2026-03-17

## 1. 当前起点

当前主工程：`e2studio 工程 Robotic_arm`（FSP 6.3.0，裸机）

已实现能力：

- I2C 主机驱动（IIC1，P511/P512，0x40）
- PCA9685 初始化与 PWM 输出
- 6 关节参数表（枚举、脉宽范围、安全限位、反向标志）
- 角度→脉宽转换（支持 180°/270° 差异处理）
- HOME 安全姿态自动执行
- I2C 中断回调

待补充能力：

- UART 日志输出（当前无任何调试信息）
- 多姿态定义与切换（PRE_GRASP/GRASP/LIFT/PLACE）
- 按键触发姿态切换
- 异常停机状态机
- 串口命令解析

## 2. 下一步的软件结构

建议在现有工程基础上逐步形成下面的层次：

### 板级基础层

负责：

- UART 日志输出
- 板载 LED 状态提示
- KEY 输入与急停复用预留
- 错误处理和状态码打印

### 驱动板控制层

负责：

- 向舵机驱动板发送控制量
- 统一管理单路舵机输出
- 提供最小接口：
  - `driver_board_init()`
  - `driver_board_set_channel_pulse(channel, pulse_us)`

### 舵机抽象层

负责：

- `joint_id -> channel`
- 角度限制
- 角度到脉宽映射
- 180 度与 270 度差异处理

最小接口建议：

- `servo_init()`
- `servo_set_angle(joint_id, angle_deg)`
- `servo_set_safe_pose()`

### 动作层

负责：

- 固定点位切换
- 简单状态机
- 单步调试动作

最小动作集合：

- `robot_pose_home()`
- `robot_pose_ready()`
- `robot_pose_pick()`
- `robot_pose_lift()`
- `robot_pose_place()`

## 3. 关节配置建议

每个关节都要建立参数表，至少包含：

- `joint_id`
- 驱动板通道号
- 舵机类型
- 最小安全角度
- 最大安全角度
- 零位角度
- 中位对应脉宽
- 最小脉宽
- 最大脉宽

底座舵机单独建表，不与普通 180 度舵机共用映射。

## 4. 推荐推进顺序

### 第一步

仅接 1 个 180 度舵机，完成：

- 初始化
- 中位保持
- 小幅度往返
- 串口打印目标角度和脉宽

### 第二步

接入 270 度底座舵机，完成：

- 独立映射
- 独立安全范围
- 不与其他关节共用参数

### 第三步

接 2 到 3 个关节，完成：

- 顺序动作
- 同步动作
- 回到安全位

### 第四步

扩展到整臂，完成：

- 关键位姿表
- 状态切换
- 固定点抓取流程

## 5. 日志建议

UART 在现阶段保持为唯一调试主通道，日志内容至少包含：

- 上电完成
- 驱动板初始化结果
- 单关节目标角度
- 输出脉宽
- 当前动作名
- 错误码
- 急停或异常状态

## 6. 这一阶段先不要做的事

- 不先做 TinyML
- 不先做复杂串口协议
- 不先做完整逆运动学
- 不先做多目标视觉调度
- 不先让 ADC 压力采样和执行链路同时并行开发
