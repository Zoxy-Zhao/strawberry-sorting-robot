# 操作流程手册

## 1. 新建 RA6M5 下位机工程

目标工程：

- `D:\e2studio_test\Robotic_arm`

流程：

1. 在 e2studio 新建 `Renesas RA C/C++ Project`
2. 芯片选择 `R7FA6M5BF2CBG`
3. 选择 `FSP + Bare Metal`
4. 工程位置选 `D:\e2studio_test`
5. 工程名填 `Robotic_arm`

## 2. FSP 最小配置

`Stacks`：

- `g_ioport`
- `g_i2c_master1`

`g_i2c_master1` 参数：

- `Channel = 1`
- `Rate = Standard`
- `Slave Address = 0x40`
- `Address Mode = 7-bit`
- `Callback = i2c_master1_callback`
- `IPL = 12`
- `Timeout during SCL Low = Enabled`

`Pins`：

- `P511 = IIC1_SDA1`
- `P512 = IIC1_SCL1`

## 3. 首轮接线

逻辑侧：

- `P511 -> SDA`
- `P512 -> SCL`
- `3V3 -> VCC`
- `GND -> GND`

动力侧：

- PCA9685 舵机电源接外部降压模块
- 外部电源地与 RA6M5 地共地

## 4. 首轮验证流程

目标：先让 `CH0` 回中位。

流程：

1. 在 `hal_entry.c` 中写入最小版本
2. 烧录
3. 舵机接 `CH0`
4. 上电观察是否回到 `1500us` 中位

## 5. 单舵机标定流程

目标：测出每个通道的 `left_limit_us` 和 `right_limit_us`。

流程：

1. 只接一个舵机
2. 固定 `SERVO_CENTER_US = 1500`
3. 逐步测试目标脉宽
4. 记录左右极限
5. 暂时统一采用保守的 `safe_us`

当前经验值：

- `left_safe_us = 500`
- `right_safe_us = 2750`

## 6. 多舵机推进顺序

建议顺序：

1. `CH0` 单舵机
2. `CH1` 单舵机
3. `CH2`
4. `CH3`
5. `CH4`
6. `CH5`
7. 6 舵机 `HOME` 姿态
8. 5 姿态切换
9. 固定点假草莓抓取

## 7. 现阶段代码推进顺序

建议从易到难：

1. `I2C + PCA9685 + 单舵机中位`
2. `单舵机角度映射`
3. `6 舵机参数表`
4. `HOME` 安全姿态
5. `PRE_GRASP / GRASP / LIFT / PLACE`
6. 串口命令触发姿态
7. 固定点抓取状态机

## 8. 烧录和供电建议

- RA6M5 优先由电脑 USB 供电
- 舵机由外部电源供电
- 烧录时逻辑侧接线可保持不变
- 若仅排查烧录问题，可先不上舵机动力电源

## 9. 故障判断

如果 `pyocd` 提示：

`Waiting for a debug probe matching unique ID ...`

优先排查：

- 调试探头 UID 是否写错
- USB 线是否支持数据
- 板子是否真正上电
- 调试探头是否被 pyOCD 识别
