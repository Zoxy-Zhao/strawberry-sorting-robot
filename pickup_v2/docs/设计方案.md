# 任意位置草莓抓取 — 设计方案

> 本文档是本次升级的**主设计文档**，覆盖架构总览、坐标系链路、阶段划分、隔离策略、明确不做项。
> 详细的算法推导、操作手册、协议格式分别在同目录下的其它文档。

## 1. 目标

把现有的「Pi 检测分类 → MCU 走固定姿态序列 PRE_GRASP/GRASP/LIFT/PLACE」流程，升级为：

> Pi 检测到草莓 → 计算其在机械臂坐标系下的 (X, Y, Z) → 解析逆运动学 → 串口发关节角 → MCU 移臂抓取 → 按分类放置

**核心变化：抓取位置不再是固定的 PRE_GRASP，而是根据视觉实时计算。**

## 2. 关键决策（已与用户确认）

| 决策点 | 选择 | 理由 |
|---|---|---|
| 工作模式 | 停下来抓（静态） | 难度低，5mm 精度可达，无时序问题 |
| 摄像头方案 | 单目 eye-to-hand | 传送带平面假设 → 单应矩阵足够；不需第二个相机 |
| IK 计算位置 | Pi 端用 numpy | 开发快、易调试、首版优选 |
| 末端姿态 | 夹爪垂直向下抓 | 简化 IK 为 4 自由度闭式解；草莓朝向暂不判断 |
| MCU 改动方式 | 完整副本 `hal_entry_pickup_v2.c` | 不污染现有代码，可手动决定是否替换 |
| 隔离粒度 | 全部进 `pickup_v2/` 目录 | 删目录即回滚，零侵入 |

## 3. 总体架构

### 3.1 系统数据流

```
[CSI 摄像头]
    │
    ↓
[YOLO 检测 → bbox 中心像素 (u, v)]
    │
    ↓ ① 内参去畸变
[去畸变像素 (u', v')]
    │
    ↓ ② 单应矩阵 H（假设草莓在传送带平面上）
[工作面坐标 (Xw, Yw, 0) mm]
    │
    ↓ ③ 平移变换 T（工作面原点 → 臂基原点）
[臂基坐标 (Xa, Ya, Za) mm]
    │
    ↓ ④ 解析逆运动学 IK
[6 个关节角 (θ5, θ1, θ0, θ2, θ3, θ4) deg]
    │
    ↓ ⑤ 串口协议 "K θ5 θ1 θ0 θ2 θ3 θ4\n"
[MCU 接收 → robot_apply_pose_ex 插值移动]
    │
    ↓
[6 路 PWM → PCA9685 → 舵机]
```

### 3.2 模块职责

| 模块 | 文件 | 职责 |
|---|---|---|
| 标定 | `calibration/intrinsic_calib.py` | 棋盘格法求相机内参 + 畸变系数 |
| 标定 | `calibration/homography.py` | 工作面 4-9 点对求单应矩阵 H |
| 坐标 | `pi/coord_transform.py` | 像素 → 工作面 → 臂基坐标三步变换 |
| 算法 | `pi/kinematics.py` | 解析 IK、FK 反算、工作空间检查 |
| 协议 | `pi/protocol_v2.py` | 串口命令封装（M/K/J/OPEN/CLOSE/HOME）+ ACK 等待 |
| 主控 | `pi/main_pickup.py` | YOLO → 坐标 → IK → 协议的总调度 |
| MCU | `mcu/hal_entry_pickup_v2.c` | 行缓冲解析 + 兼容旧单字符协议 + 复用 servo_set_angle |

## 4. 4 层坐标系（详见 `坐标系与IK.md`）

| 坐标系 | 单位 | 原点 | X 轴方向 | Y 轴方向 | Z 轴方向 |
|---|---|---|---|---|---|
| 图像坐标 | px | 左上角 | 向右 | 向下 | — |
| 工作面坐标 | mm | 标定时人为指定（建议传送带某固定标记点） | 沿传送带前进方向 | 与 X 垂直，向相机左侧 | 向上（不用，恒为 0） |
| 臂基坐标 | mm | 机械臂底座旋转轴中心 | 朝臂初始指向（HOME 朝前） | 右手系定 Y | 向上 |
| 关节角 | deg | — | — | — | — |

**关键参数（已知）：**
- 连杆：H1=95mm, L2=130mm, L3=140mm, L4=45mm, L5=55mm
- 舵机标定数据：`docs/舵机标定记录.csv`（每个关节 reversed/safe_us 表）

## 5. 隔离策略

### 5.1 现有项目零修改

| 现有路径 | 状态 |
|---|---|
| `vision/pi/main.py` | 不动 — 旧的分类版本继续可用 |
| `vision/pi/config.py` | 不动 |
| `vision/pi/camera.py` `detector.py` `serial_comm.py` | 不动（新代码 import 复用） |
| `docs/*.md` | 不动 |
| `mcu/hal_entry.c` | 不动 — 由你手动决定是否替换为 v2 |

### 5.2 新协议与旧协议平行

旧版 hal_entry.c 收到新命令（如 `M 100 50 20\n`）会进 `process_pi_char` 的 default 分支，逐字符报 `Unknown command`，**不会崩溃**。这意味着：

- 旧 hal_entry.c + 旧 main.py：分类模式正常运行
- 旧 hal_entry.c + 新 main_pickup.py：失败（MCU 不识别 M 命令），但不损伤设备
- 新 hal_entry.c + 旧 main.py：分类模式仍正常运行（兼容旧单字符协议）
- 新 hal_entry.c + 新 main_pickup.py：完整任意位置抓取功能

### 5.3 配置不互相 import

`pi/config_v2.py` 完全独立，不 `from config import ...`。即使旧 config.py 被改了，新版本也不受影响。

## 6. 阶段划分与验收

| 阶段 | 内容 | 验收标准 | 估时 |
|---|---|---|---|
| **A：标定模块** | 拷代码 + 跑相机内参 + 单应矩阵 | RMS<1px，单应平均误差<5mm | 1 天 |
| **B：IK 算法** | Python 实现 + pytest | FK(IK(p))≈p 误差<0.1mm，9 网格点全过 | 1 天 |
| **C：MCU v2** | 生成 `hal_entry_pickup_v2.c` + 烧录 + `K` 命令调试 | `K 135 70 70 150 80 120` 能让臂回 HOME | 1.5 天 |
| **D：Pi-MCU 联调** | 协议串通 + 干跑 | 输入像素 → 臂尖停目标上方 5mm | 1 天 |
| **E：实物抓取测试** | 9 网格点放橡皮球 / 草莓 | 抓取成功率 ≥ 7/9 | 1.5 天 |
| **F（可选）：调优** | 误差超 10mm 时回查 | 全工作面定位误差 ≤ 8mm | 1 天 |

**总工期：5-7 天**（按学生兼顾课业节奏，可拉至 2 周）

## 7. 系统级验收

1. 草莓放在工作面任意位置 → 系统自动定位 + 抓取成功
2. 单次抓取周期 < 8 秒（检测+IK+移动+夹+抬起）
3. 工作空间外目标 → 报 `NACK UNREACHABLE`，不飞臂
4. 任何环节失败 → 自动回 HOME，不卡死

## 8. 明确不做（YAGNI）

- ❌ 末端二次视觉微调（一期不做，靠 V 型夹爪+软指吃误差）
- ❌ 草莓朝向自动判断（默认垂直抓）
- ❌ 多目标调度（一次抓一个）
- ❌ 带跟踪（已确认停下来抓）
- ❌ 障碍规避（工作面假设干净）
- ❌ 路径规划（直接走插值，MCU 现有 TRANSIT 机制够用）
- ❌ 替换现有的 PLACE 姿态系统（PLACE_A/B/C 仍走原表）

## 9. 风险与兜底

| 风险 | 兜底 |
|---|---|
| 单应矩阵误差 5-10mm | V 型夹爪 + 软指（机械吃误差） |
| 摄像头位姿漂移（运输后） | 现场 5 分钟重标定 SOP（只重做单应矩阵） |
| IK 无解 | Pi 端先检查工作空间，越界直接拒绝 |
| 关节角超限 | MCU 端 servo_angle_to_us 已有 clamp |
| 突发动作（关节角跳变 >60°） | Pi 端单调性检查 + MCU 插值缓动 |
| 抓取力过大碰坏草莓 | 复用现有 FSR 力反馈逻辑（已就绪） |

## 10. 与现有项目的复用

新代码大量复用现有模块（import 不改）：

- `vision/pi/detector.py` → YOLO 推理
- `vision/pi/camera.py` → CSI 摄像头封装
