# 验收标准

> 每个阶段交付物的验收清单。**未达标不进下一阶段。**

## 阶段 A：标定模块（1 天）

| 项 | 验收标准 | 验证方法 |
|---|---|---|
| 内参 RMS | < 1.0 px（< 0.5 优秀） | `intrinsic_calib.py compute` 输出 |
| 去畸变 | 直线变直 | 肉眼对比 |
| 单应矩阵平均误差 | < 5 mm（< 8 mm 可接受） | `homography.py fit` 输出 |
| 单应矩阵最大误差 | < 8 mm | 同上 |
| 工作面 → 臂基偏移 | 端到端验证误差 < 8 mm | 已知物体位置对照 |
| 标定文件齐全 | 5 个 yaml/csv 文件存在 | `ls outputs/` |

## 阶段 B：IK 算法（1 天）

| 项 | 验收标准 | 验证方法 |
|---|---|---|
| FK / IK 自洽 | `fk(ik(p)) - p` 误差 < 0.1 mm | `pytest test_kinematics.py` |
| 9 网格点全过 | 工作空间内 9 个测试点都能求解 | 同上 |
| 边界检测 | 不可达点抛 `IKUnreachableError` | 单元测试 |
| 单调性检测 | 关节差 > 60° 时抛警告 | 单元测试 |
| 数值稳定 | 边界 acos / sqrt 不出 NaN | 单元测试覆盖 cos = ±1 边界 |
| 已知姿态对照 | 现有 PRE_GRASP / GRASP 用 FK 算的位置与 IK 反算一致 | 手动比较 |
| 关节角→舵机us | 9 个 K 命令测试值与示教模式记录一致 | 标定数据回归测试 |

## 阶段 C：MCU v2（1.5 天）

| 项 | 验收标准 | 验证方法 |
|---|---|---|
| 编译通过 | e2studio 无 error，warning ≤ 原工程 + 0 | 编译输出 |
| 烧录成功 | pyocd flash 完成，复位后 LED 行为同原版 | 上电观察 |
| 旧协议兼容 | 发 `G`（不带 \n）正常启动传送带 | PuTTY 手动测 |
| 旧协议兼容 2 | 发 `A`/`B`/`C` 仍走原 PRE_GRASP/GRASP 流程 | 同上 |
| HOME 命令 | 发 `HOME\n` 回 `READY\n` 且臂回零 | 同上 |
| 单关节命令 | `J 5 90\n` 转底座到 90° 准确 | 量角器 |
| 完整 K 命令 | `K 135 70 70 150 80 120\n` = 现有 PRE_GRASP | 目测姿态对比 |
| 错误处理 1 | `K 9999 0 0 0 0 0\n` 回 `NACK SAFETY\n` | 同上 |
| 错误处理 2 | `BLAH\n` 回 `NACK BADARG\n` | 同上 |
| 急停仍工作 | `X` 仍能急停 | 同上 |
| 示教模式仍工作 | 调试 UART `T` 进示教，方向键正常 | 同上 |
| FSR 仍工作 | 抓取时 FSR 反馈正常 | 同上 |
| TinyML 仍工作 | 抓取时 TinyML 输出 STABLE / SLIP_RISK | 同上 |

## 阶段 D：Pi-MCU 联调（1 天）

| 项 | 验收标准 | 验证方法 |
|---|---|---|
| 协议封装可用 | `ProtocolV2` 类发命令收响应正常 | pytest 模拟串口 |
| 端到端坐标转换 | 已知像素 (320, 240) → 算出臂基 (X, Y, Z) 与手测一致 | 干跑日志 |
| 干跑模式定位 | 把 5 个测试目标贴在工作面，--dry-run 输出的目标坐标都在物体附近 ±8mm | 量距 |
| MCU 移臂目测 | `M X Y Z+30` 后臂尖在目标上方 30mm 处，肉眼对齐 | 拉直尺测 |
| 异常恢复 | 发不可达坐标后 Pi 报错且 MCU 不动作 | 故意发越界 |
| 全流程时序 | 从 G → 检测 → M → CLOSE → PLACE → HOME 一遍跑完，无超时 | 主程序日志 |

## 阶段 E：实物抓取测试（1.5 天）

### E.1 网格测试（替代物：橡皮球 / 小番茄）

把 9 个目标放在工作面 9 个网格位置（3×3 均匀分布）：

| 测试 | 目标 | 通过条件 |
|---|---|---|
| 中心点 | 1 个目标在工作面正中 | 抓起 |
| 4 角 | 4 个目标在 4 角 | 至少 3/4 抓起 |
| 4 边中点 | 4 个目标在 4 边中点 | 至少 3/4 抓起 |
| **总成功率** | — | **≥ 7/9** |

### E.2 真草莓测试

| 测试 | 通过条件 |
|---|---|
| 单颗成熟草莓任意位置 | 5/5 抓起且不夹烂 |
| 半成熟 / 未熟草莓 | 抓起后正确放入对应分类筐 |
| FSR 反馈生效 | 抓取力适中，软草莓不损伤 |

### E.3 异常场景

| 场景 | 期望行为 |
|---|---|
| 草莓在工作空间边缘外 | Pi 报 UNREACHABLE，MCU 不动作 |
| 检测到草莓但抓空 | 不影响下一次循环 |
| 闭爪过程中 FSR 异常 | TinyML / FSR 触发 SLIP_RISK，记录日志 |
| Pi 与 MCU 通信中断 | Pi 端超时报警，3 秒后自动重发 G\n |

## 阶段 F（可选）：调优（1 天）

只在 E 阶段成功率 < 7/9 时启动。

| 排查方向 | 工具 |
|---|---|
| 哪一层误差大 | `pi/coord_transform.py` 增加 verbose log，逐层打印中间坐标 |
| 是 IK 误差还是机械误差 | 给 K 命令直接发 PRE_GRASP 角度，看实际位置是否准 |
| 是单应矩阵误差 | 重做阶段 A2，加密标定点 |
| 是偏移测量误差 | 重新量直尺 |
| 是抓取点 Z 误差 | 调整 strawberry_grasp_z_mm |
