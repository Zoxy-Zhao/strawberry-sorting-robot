## 1. 背景：pickup_v2 要哪些标定

把抓取从"固定姿态序列"升级为"视觉定位 + IK + 任意位置抓取"，需要 5 次标定：
- **E1 关节标定**：几何角(IK输出) ↔ 舵机角。本体校准，本文档主角。
- E3 相机内参 / E4 单应 / E5 臂基偏移：视觉感知校准（本次未做）。

硬件关键约束：**舵机是开环 PWM（PCA9685 驱动，无位置反馈）**——这是后面所有麻烦的根源。

## 2. E1 标定的演进历程（核心：踩了哪些坑、怎么一步步逼近正确）

### 阶段 1：2 点法 + 强制 SIGN=±1（最初设计）

- **模型**：`servo = OFFSET + SIGN×geom`，每关节标 2 点（几何 0° 和另一个角），SIGN 只取 ±1（假设 1 舵机度=1 几何度，即理想直驱）。
- **实测结果**（4 关节）：

| 关节 | 点1 | 点2 | 残差 |
|---|---|---|---|
| base | 0°→servo127 | 45°→172 | 0° |
| shoulder | 0°→37 | 45°→73 | **9°** |
| elbow | 0°→163 | 90°→85 | **12°** |
| wrist | 0°→90 | −45°→135 | 0° |

- **发现**：shoulder/elbow 残差 9°/12°，说明它们**不是 1:1**。base/wrist 残差 0（但部分因第二点是按 1:1 直接填的）。

### 阶段 2：升级"真实斜率"（不再强制 ±1）

- **改动**：用两点真实斜率 `slope=(s2−s1)/(g2−g1)`，不再只取符号。
- **实测斜率**：base **1.00** / shoulder **0.80** / elbow **−0.87** / wrist **−1.00**。
- **根因锁定 = 重力下垂**，且数据自洽：
  - base 绕**竖直轴**转，重力无力矩 → 1:1 ✅
  - shoulder **受力矩最大**（扛整条臂）→ 下垂最多 → 斜率最小 0.8
  - elbow 受力中等 → 0.87
  - wrist 末端最轻 → ≈1:1
  - 物理解释：水平时重力力矩最大、舵机要多使劲顶住，抬高后力矩变小，导致"舵机走的量 < 几何角变化量" → 斜率<1。

### 阶段 3：反推 φ_fixed 时，暴露"线性外推"致命缺陷

- 为改 IK（垂直够不到→固定斜角），需把老工程成功抓取姿态 `g_pose_grasp` 的舵机值换算成几何角。
- shoulder 在抓取姿态用到 **servo 130**，远超标定范围(0~45)。
- **线性外推**：geom = (130−37)/0.8 = **116°**；**实测只有 85°**（大臂离垂直 5°）。**差 31°！**
- **结论**：重力下垂是**非线性**的——高角度时手臂更竖、下垂变小、斜率回升趋近 1。0~45° 标的线性斜率(0.8)外推到操作角(85°)严重失真。**必须在操作角度区间标定。**

### 阶段 4：多点分段线性插值（最终方案）

- **改动**：每个受力关节存 **N 个标定点**，几何→舵机用**分段线性插值**（区间外端段斜率外推）。
- **覆盖范围**（由仿真扫描 10×10cm 取草莓区算出操作范围）：shoulder 53~111°、elbow 37~112°。
- **目标标定点**：shoulder = 50/70/90/110°；elbow = 40/65/90/115°（base/wrist 已确认 1:1，保持 2 点不动）。
- 状态：代码已改完并测试通过；**实物 4 点重标待执行（2026-06-04）**。

## 4. φ_fixed 的确定与验证方法（可复用的验证套路）

**目标**：垂直朝下(−90°)够不到传送带，改"固定斜角 φ_fixed"。φ_fixed = 末端世界俯仰角。

**确定**：取老工程 `hal_entry.c:185 g_pose_grasp` 舵机 {126,130,91,135,80,65}（实物发过去确认能夹到草莓、版本无误）→ 换算/实测几何角 → φ_fixed = α_s−α_e+α_p = 85−83−45 = **−43°**。实测末端对地面也 ≈ −43°，吻合。

**验证套路（强烈推荐复用）**：`sim/verify_grasp_roundtrip.py`
```
已知能用的真实姿态角 --FK--> 末端位置(x,y,z) --新IK(φ_fixed)--> 解出角度
若解出角度 ≈ 原始角度 → IK 数学 + φ_fixed 正确
```
- 结果：FK 得草莓位置 (301.5, −5.3, 97.4)，IK 解回 shoulder85/elbow83/wrist−45，**差 0.00**。
- **妙处**：把"IK 数学对不对"和"标定准不准"**分离**开。roundtrip 在几何角空间完美闭合 → IK 正确；而 geom→servo 那一步 shoulder 差 25°（IK要85°→E1线性给舵机105，实际需130）→ 问题定位在标定，不是 IK。

## 6. 可复用经验教训（给未来类似项目）

1. **开环 PWM 舵机有重力下垂 → 舵机角↔几何角是非线性、且随姿态/负载变**。屏幕显示的舵机值是"发出去的指令"，不是"真实到达的角度"——必须用量具实测核对。
2. **一定在"实际操作角度区间"标定，不要在方便的区间（如 0~45°）标完外推**。本项目操作角在 85° 附近，0~45° 标的线性外推差了 31°。
3. **2 点线性只够用于"无重力力矩(竖直轴)或轻负载"的关节**（base/wrist）；受力关节(shoulder/elbow)要**多点分段**。
4. **平面传送带抓取用"固定末端朝向 φ_fixed"的 IK 最简且够用**；φ_fixed 从一个"已验证能抓"的真实姿态反推最可靠。
5. **FK→IK 闭环 roundtrip 是分离"算法 bug"和"标定误差"的利器**——锚点用一个真实可用的姿态。
6. **想根治下垂猜测 → 换带编码器的总线舵机**（能读回真实角度），代价是重做下位机驱动，留给未来新臂。
7. 验证要在改硬件/搬运后重做：E3 只标一次，E4/E5 相机或臂或传送带一动就失效。

### ⚠ 已知遗留：shoulder 几何→舵机 14° config 依赖缺口

**现象**：新 4 点标定算 几何85°→servo**116**；但抓取姿态实测锚点是 servo**130**。差 **14°**。

**根因**：开环舵机重力下垂**看 elbow 负载/姿态**。标 shoulder 时 elbow 锁在 HOME(servo70，约 geom102°)，而真实抓取时 elbow 折角不同 → 肩部力矩不同 → 同 servo 下垂量不同。**这是开环臂的固有局限，纯标定数据无法消除**（复现根因：换带编码器总线舵机才能根治，见第 6 节教训 #6）。

**为什么不现在修**：单一 elbow config 下标的 shoulder 曲线，无法对所有抓取姿态都准。强行在抓取 config 标，又只对那一个抓取点准。**正确做法是用 E2 实测量化真实误差再决定补偿策略**。

**E2 处置预案**（误差<8mm 验收）：
- 若肩**系统性偏低**（指尖落点比目标矮/近）→ 优先方案 A：加一个**与 elbow 角相关的 droop offset**（查表或线性补偿）。
- 退而求其次方案 B：用**接近典型抓取的 elbow 角**重标 shoulder（牺牲非抓取姿态精度）。
- 若误差已 <8mm（下垂在该工作点恰好被 4 点曲线吸收）→ 不动，记录即可。

## 8. E2 实测结论（2026-06-05）：双关节下垂 + 甜点带策略

**E2 用 `calibration/e2_probe.py`（坐标→IK→K→量落点）实测，得出颠覆性结论。**

### 8.1 关键实测数据（z=97, y=0）

| r | 指令肩角 | 实测 | 误差 | 主垮关节 |
|---|---|---|---|---|
| 270 | 101° | Z=−10（垮107mm）| 大 | **肩** |
| **300** | 86° | Z=90≈97 | **3mm ✅** | 几乎不垮 |
| 350 | 59° | 肩实50°/肘α_e实74°(指令51) → FK落点(267,−51) | X短83+Z垮148 | **肘(小臂)** |

> r350 实测关节角(水平仪/角度尺)：大臂50°、大小臂夹角106°(→α_e=180−106=74)。位置卷尺读数(180,345)不可靠，**以角度读数为准**(量角比量3D位置准)。

### 8.2 真正结论：开环下垂是全工作区、分布在多关节、随姿态变

- **近点(r小)** → 肩角抬高(>100°) → **肩垮**。
- **远点(r大)** → 手臂伸远 → **肘(小臂)垮**（小臂在重力下多折）。
- **只有中间 r≈300(肩~86/肘~84)** 两关节力矩都小 → 准(3mm)。这是一个**特定甜点姿态**，不是某个肩角阈值。
- 之前怀疑的"14° 肩缺口"被推翻：r300 的 Z 准说明肩没大垮；那个 servo130 锚点不可靠。
- **"垫高底座"思路否决**：垫高让肩角降，但低肩远伸又换成肘垮，治标不治本。

### 8.3 决策（2026-06-05）：限定甜点带演示

**草莓只放在 r≈285~320 的圆弧带内**（IK 直接准，~3mm），放弃全区任意位置。理由：双关节下垂无法用单标定消除，补偿需多关节建模且近饱和不稳；演示求稳。

- **下垂只跟 r 有关、与 y(方位)无关**（重力竖直，base 绕竖直轴转不改力矩）→ 甜点带是**圆弧带**(某 r 范围 × 整个 y 范围)，沿 r 标一次即可。
- 待办：量 r=285/315/330 关节角，FK 算误差，定死带宽边界。
- 草莓放带内 → 接 E3/E4/E5 相机标定。E2 看到的 X/Y 偏(45,−25)是**量尺原点没对准真旋转轴**，E5 臂基偏移会自动吸收，非臂误差。
- 根治(覆盖全区) = 换编码器/总线舵机(能读回真实角)，留未来新臂。

### 8.4 取料线验证通过（2026-06-05）

- **准的半径 ≈ r300，与抓取高度 z 无关**（z=97/85/72/60 算下来紧凑准姿态都在 r≈300）。
- **实测验证**：(300,0,72) 与 (296,50,72)（等 r=300）两点 Z 都≈72、彼此一致、肩≈85° → **真实抓取高度下准 + y-自由成立**。
- **运行规格锁定**：取料半径 **r≈300**、抓取高度 **z≈65**（z=72 实测稍高，夹草莓需~65；7mm 差姿态几乎不变，准度延续）、**y 自由**。
- **硬约束**：准区仅 r≈295~305（~10mm 宽，r315 就垮 67mm）。传送带深 100mm，**须用导轨把草莓约束到 r≈300±10mm 这条"取料道"**（夹草莓中心即可，30~40mm 直径草莓边缘超出准区无妨）。
- 传送带几何：深(X)100mm[近边 r≈295/中心 r≈345/远边 r≈395]、长(Y)1000mm、左→前送、抓时停。下垂与 y 无关→正前方那段任意 y 都准。
- 摆法待定：甲(臂不动+草莓近边道 r300+导轨) / 乙(臂前移~45mm 使中心=r300)。定了才做相机标定(依赖最终几何)。

