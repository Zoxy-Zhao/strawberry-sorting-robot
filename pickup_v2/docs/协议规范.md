# 串口协议规范 v2

> Pi ↔ MCU 串口通信协议。在保留旧协议（A/B/C/G/X 单字符）的基础上，新增带参数的命令以支持任意位置抓取。

## 1. 物理层

| 项目 | 值 |
|---|---|
| 接口 | SCI9（MCU 端）/ `/dev/serial0`（Pi 端） |
| 引脚 | P109 (TXD9) / P110 (RXD9) — 与现有保持一致 |
| 波特率 | 115200 |
| 数据位 / 停止位 / 校验 | 8 / 1 / N |

物理连接不变，新协议在现有连接上多路复用。

## 2. 行结束符约定

**新版本协议：所有命令以 `\n` (0x0A) 结尾。**

- Pi → MCU：每条命令末尾必带 `\n`
- MCU → Pi：每条响应末尾必带 `\n`
- `\r` (0x0D) 兼容存在但被 MCU 忽略
- MCU 端最大行缓冲：64 字节，超长直接清空并返回 `NACK BADARG`

## 3. 旧协议兼容（保留）

旧 `vision/pi/main.py` 发送的不带 `\n` 单字符命令仍然工作。新版 hal_entry_pickup_v2.c 处理逻辑：

```
收到字符 → 累积到行缓冲
  ├ 若是 \n：解析整行（新协议路径）
  ├ 若是 \r：忽略
  └ 若是 legacy 单字符且当前行缓冲为空：立即处理（旧协议路径）
```

**legacy 截胡条件**（避免与扩展命令字头冲突，2026-05-15 修复）：

| 字符 | 触发 legacy 的状态 | 备注 |
|---|---|---|
| `X` | 任意 | 急停永远优先 |
| `G` | `STATE_IDLE` | 其他状态进入行缓冲，单独的 `G\n` 会回 `NACK BADARG` |
| `A` / `B` / `C` | `STATE_BELT_RUN` | **`C` 在其他状态进入行缓冲**，否则会吞掉 `CLOSE\n` 开头 |

这意味着：
- 老 `vision/pi/main.py` 流程（BELT_RUN 时发 ABC、IDLE 时发 G）完全兼容
- 新协议 `CLOSE\n` / `HOME\n` 等在非 BELT_RUN 状态下解析正确
- 发 `b"G"` 单字节在 IDLE 立即生效；发 `b"G\n"` 任何时候都尝试解析

新版 main_pickup.py **统一带 `\n`**。

## 4. 命令字总览（Pi → MCU）

| 命令 | 格式 | 含义 | 状态机要求 |
|---|---|---|---|
| `G\n` | 单字符 | 启动传送带（兼容旧） | STATE_IDLE |
| `X\n` | 单字符 | 急停（兼容旧） | 任意 |
| `A\n` `B\n` `C\n` | 单字符 | 旧分类抓取（兼容旧） | STATE_BELT_RUN |
| `M X Y Z\n` | 三个 float (mm) | 移臂到工作面坐标 (X, Y, Z) — Pi 已算好 IK | STATE_IDLE / STATE_BELT_STOP |
| `K θ5 θ1 θ0 θ2 θ3 θ4\n` | 六个 float (deg) | 直接发关节角（调试 / IK 失败兜底） | STATE_IDLE |
| `J ch angle\n` | int + float | 单关节调试，例 `J 5 90\n` 转底座到 90° | STATE_IDLE |
| `OPEN\n` | 关键字 | 夹爪全开 | 任意 |
| `CLOSE\n` | 关键字 | 夹爪闭合（带 FSR 反馈） | 任意 |
| `HOME\n` | 关键字 | 回 HOME 姿态 | STATE_IDLE / STATE_RETURN |
| `PLACE A\n` | 关键字 + 字母 | 走原 PLACE_A/B/C 姿态（不经 IK） | STATE_IDLE |

## 5. 响应（MCU → Pi）

| 响应 | 含义 |
|---|---|
| `READY\n` | 上一命令已执行完毕 |
| `BELT_ON\n` | 传送带已启动（响应 `G`） |
| `BELT_OFF\n` | 传送带已停止 |
| `STOPPED\n` | 急停已触发（响应 `X`） |
| `BUSY\n` | 当前状态机不允许该命令 |
| `NACK UNREACHABLE\n` | 目标越界（仅 `M` / `K`） |
| `NACK BADARG\n` | 参数解析失败 |
| `NACK SAFETY\n` | 关节限位 / FSR 异常 |
| `NACK TIMEOUT\n` | 内部动作超时 |

## 6. 完整抓取时序

```
Pi → MCU         MCU → Pi          说明
─────────────────────────────────────────
G\n            ─→
              ←─ BELT_ON\n         传送带启动
                                   [YOLO 检测到草莓 → 计算 (Xa, Ya, Za)]
X\n            ─→
              ←─ BELT_OFF\n
HOME\n         ─→
              ←─ READY\n           回零（保险动作）
OPEN\n         ─→
              ←─ READY\n
M Xa Ya Za+30\n ─→
              ←─ READY\n           移到目标上方 30mm（预抓取）
M Xa Ya Za\n   ─→
              ←─ READY\n           下降到抓取点
CLOSE\n        ─→
              ←─ READY\n           带 FSR 闭爪
M Xa Ya Za+50\n ─→
              ←─ READY\n           抬起 50mm
PLACE A\n      ─→                  按分类放置（走原姿态）
              ←─ READY\n
HOME\n         ─→
              ←─ READY\n           回零
                                   [循环或停止]
```

**关键点：**
- 每条命令必须等到 `READY` 或 `NACK` 才发下一条（**严格请求-响应**）
- 超时阈值：移臂类命令 8 秒，HOME 5 秒，OPEN/CLOSE 3 秒
- 超时未响应 → Pi 端发 `X\n` 急停

## 7. 数值范围与精度

| 参数 | 范围 | 精度 |
|---|---|---|
| `M` 坐标 X | 臂基系下 0 ~ 320 mm（最大伸展 L2+L3+L_end） | 0.1 mm |
| `M` 坐标 Y | -200 ~ 200 mm（受底座旋转 270° 限制） | 0.1 mm |
| `M` 坐标 Z | -50 ~ 200 mm（不能低于地面） | 0.1 mm |
| `K` 关节角 | 各关节 safe_us 对应范围 | 0.1° |
| `J` 通道号 | 0~5 整数 | — |

**Pi 端发命令前自查范围；超出 → 不发，本地报错。MCU 是第二道防线。**

## 8. 错误处理流程

```
Pi 发 M 命令
  │
  ├ Pi 端预检（IK 求解）
  │    ├ UNREACHABLE → 不发命令，本地报警
  │    └ OK → 转关节角，发 K（或仍发 M，由 MCU 转）
  │
  ↓
MCU 接收 → 行解析
  │
  ├ 解析失败 → NACK BADARG
  ├ 状态机不允许 → BUSY
  ├ 关节超限 → NACK SAFETY
  └ 执行成功 → READY
```

## 9. 实现文件对照

| 模块 | 文件 |
|---|---|
| Pi 端协议封装 | `pi/protocol_v2.py` — class `ProtocolV2`，方法 `send_M(x,y,z)`, `send_K(*joints)`, `send_open()`, `send_close()`, `send_home()`, `wait_ready(timeout)` |
| Pi 端串口底层 | 复用 `vision/pi/serial_comm.py`（不改） |
| MCU 端解析 | `mcu/hal_entry_pickup_v2.c` 中的 `parse_extended_cmd()` 函数 |
| MCU 端命令分发 | 同文件 `dispatch_extended_cmd()` |

## 10. 调试技巧

- 用 `screen /dev/serial0 115200` 或 PuTTY 手动测命令（必须手动按 Enter 发 `\n`）
- 单关节测试先发 `J 0 90` 再 `J 1 90` 等，逐通道验证标定方向
- 整体 IK 测试：`K 135 90 70 105 90 120` → 应该等同于现有 PRE_GRASP
