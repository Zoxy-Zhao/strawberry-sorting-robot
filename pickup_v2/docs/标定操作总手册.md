# 标定操作总手册（从这里照着做）

> pickup_v2 从"固定姿态"升级到"视觉任意位置抓取"，需要 5 次标定。本手册按**依赖顺序**把每一步
> 讲成可照做的清单：目的 / 工具 / 动作 / 命令 / 验收 / 完成后记录什么。
>
> 原理不在这里展开——想理解为什么，看 `sim/pipeline_sim.py`(臂 IK) 和 `sim/camera_sim.py`(视觉链路)
> 两个可视化，或 `docs/坐标系与IK.md`。本手册只管"怎么做"。
>
> 最后更新：2026-06-03

---

## 0. 全局顺序（为什么是这个顺序）

```
E1 关节标定  →  E1.5 改IK固定斜角  →  E2 单点验证  →  E3 内参  →  E4 单应  →  E5 偏移  →  E6/E7 端到端
└─── 臂本体（与相机无关）───┘                      └────── 相机感知 ──────┘
```

- **先臂、后相机**：相机标定要有干净基准。本体（E1/E2）没准，相机出了偏差你分不清是"看错"还是"动错"。
- **E1 必须最先**：要从成功夹取姿态反推固定斜角 φ_fixed，得先有 E1 的 OFFSET/SIGN 把舵机值换成几何角。
- **E3→E4 不可颠倒**：单应假设输入是去畸变后的干净像素。

| 标定 | 标一次能用多久 | 什么动了要重标 |
|---|---|---|
| E3 内参 | 永久 | 换镜头/改焦/改分辨率 |
| E4 单应 | 相机+带面不动就一直用 | 相机移动、传送带移动 |
| E5 偏移 | 臂+带面不动就一直用 | 机械臂移动、传送带移动 |

> 结论：装好后用支架/热熔胶把**相机、臂、传送带都固定死**，比赛搬运后务必重标 E4/E5。

## 1. 前置准备（开始前一次性做好）

**硬件 checklist**
- [ ] MCU 烧的是 `hal_entry_pickup_v2.c`（cfbd63c 或更新版）
- [ ] Pi ↔ MCU 串口连通（SCI9，115200）
- [ ] **拔掉 MCU 的 USB Debug 线**——接地环路会让 PWM 抖动（已知坑）
- [ ] 工作台清出至少 60×60cm 让臂自由摆动
- [ ] 臂上电（三路供电正常）

**Pi 连接与路径约定**（命令都在 Pi 上跑）
- 连接：PowerShell 里 `ssh pi`（免密已配）。连不上多半是热点 IP 变了 → 见手机热点已连设备列表拿新 IP，改 `~/.ssh/config` 的 HostName。
- Pi 工程根目录：`~/vs_code/strawberry_grasp/`（**非 git，靠 scp 同步**）
- Python 用 venv：`~/robot_env/bin/python`（无需 activate），下文命令里的 `python` 都替换成它，或先 `source ~/robot_env/bin/activate`

**工具 checklist**
- [ ] 手机数字水平仪 app（±0.5°，标 base 朝向 / shoulder 抬角）
- [ ] 数显角度尺 / 量角器（标 elbow / wrist 的关节相对角，比两次水平仪相减准）
- [ ] 直尺、卷尺、记号笔、胶带
- [ ] 棋盘格标定板（A4 打印 9×6 内角点，量准方格边长，比如 25mm）
- [ ] 可在传送带平面铺的网格纸/直尺（E4 读真实 mm 用）

## 2. 步骤 1 — E1 关节标定（现在就做这个）

**目的**：测出 base/shoulder/elbow/wrist_pitch 四个关节的 OFFSET/SIGN，即"几何角 ↔ 舵机角"换算。
**耗时**：约 20 分钟，必须在臂旁手动做。

### 2.1 核心心法（先记住，标的时候不慌）

- 模型是 `servo = OFFSET + SIGN × geom`，脚本**强制 SIGN=±1**（直驱臂 1 舵机度≈1 几何度）。
- **每个关节"几何角=0"那一点是命根子**：OFFSET 直接等于这点的舵机值。这点务必量准。
- **第二个点（45°/90°）只用来确认方向 + 算残差**，不用纠结精确命中。
- 残差是体检：某关节残差 >3° → 它可能不是干净 1:1，单独标记，找我看。

### 2.2 每个关节怎么摆、怎么量

| 关节 | 点1（几何=0，**关键**） | 点2（确认方向） | 量法 |
|---|---|---|---|
| **base** 底座 | 臂对准桌面"正前方"基准线 | 左转 45° | 桌面贴胶带做基准线，直尺量转角 |
| **shoulder** 肩 | 大臂**水平外伸**，水平仪读 0° | 大臂上仰 45° | 手机水平仪贴大臂 |
| **elbow** 肘 | 大臂小臂**共线**（一条直线）| 内角 90° | 共线目视，90° 用角度尺/直角尺卡 |
| **wrist_pitch** 腕 | 末端与小臂**共线** | 末端相对小臂下俯 45° | 用角度尺直接卡"末端与小臂"夹角 |

> 几何角约定（与 `kinematics.py` 一致）：α_s 大臂相对水平(0=水平,+90=朝上)；α_e 大小臂弯折(0=伸直)；α_p 末端相对小臂(0=共线,负=下俯)。

### 2.3 命令

```bash
cd ~/vs_code/strawberry_grasp/pickup_v2/calibration
~/robot_env/bin/python joint_calib.py all      # 依次标 base→shoulder→elbow→wrist_pitch
~/robot_env/bin/python joint_calib.py merge    # 合并成 outputs/joint_offsets.yaml
~/robot_env/bin/python joint_calib.py verify   # 跑 3 个测试姿态目测验收
```

微调循环命令：`+1 / -1 / +5 / -5 / =<num>（直接设值）/ done（满足目标，记录进下一点）/ skip（跳过该关节）/ abort（急停退出）`

### 2.4 安全

- 单步微调 **≤5°**，防飞舵；手放急停旁。
- 跑前确认各关节起始 servo 不会一上来顶到硬限位（尤其 elbow 起点 servo=0）。
- 听到异响/看到机械干涉，立刻 Ctrl+C（脚本 abort 会发急停 X）。
- **整套标定结束前别断电**，否则前序关节要重测。

### 2.5 验收

- 大臂水平时，水平仪误差 **< 3°**。
- `verify` 里"末端垂直朝下"那个姿态可能本来就够不到（马上要改斜角），目测方向大致对即可。
- 残差 < 3°。超差 → 重标该关节。

### 2.6 标定完成后记录

1. `outputs/joint_offsets.yaml` 的内容（4 关节 offset/sign + 残差）。
2. **之前能稳定夹取的那组姿态的 6 个舵机值**（用来反推固定斜角 φ_fixed）。

> 详细分步另见 `docs/关节标定SOP.md`。

## 3. 步骤 1.5 — 改 IK 固定斜角

**目的**：垂直朝下(-90°)物理够不到传送带，改成固定斜角 φ_fixed。
**为什么现在**：拿到 E1 的 OFFSET/SIGN 后，才能把你的成功姿态舵机值换算成几何角，反推 φ_fixed。

**准备**：
1. 整理 E1 产物 + 成功姿态 6 个舵机值。
2. （可选，加深理解）先用仿真预判：跑 `python sim/pipeline_sim.py`，把草莓拖到传送带实际位置，调 φ 滑块看变绿的角度，那个大概就是 φ_fixed。

**IK 改写**：算 φ_fixed → 改 `pi/kinematics.py` 3 处（r_w、z_w、workspace 检查）→ 加单元测试验证 `fk(ik())` 自洽 + 数值验算传送带位置确实可达。

---

## 4. 步骤 2 — E2 单点验证

**目的**：证明"E1 + 改后 IK"真的准，给相机标定一个干净基准。
**动作**：用 K 命令打 3~5 个**已知臂坐标**的点（比如正前方 200mm、左前方等），用卷尺/卡尺量指尖实际落点。
**验收**：误差 **< 8mm**。超差 → 回 E1 查是哪个关节偏。
**做完发我**：每个点的「目标坐标 vs 实测落点」。

---

## 5. 步骤 3 — E3 相机内参（标一次永久用）

**目的**：标出镜头内参 K 和畸变系数 dist。
**动作**：举棋盘格在镜头前换不同角度、距离、画面位置，采 **15~20 张**（角点要覆盖画面四角和中心）。

```bash
# Pi 上。CSI 摄像头若 VideoCapture 打不开，改用 capture_pi.py 抓图存进 images/
python intrinsic_calib.py capture  --camera 0 --out images/
python intrinsic_calib.py compute  --src images/ --pattern 9x6 --square 25 --out outputs/intrinsics.yaml
python intrinsic_calib.py undistort --params outputs/intrinsics.yaml --src images/ --out undistorted/
```

- `--pattern 9x6`：内角点数（不是方格数）。
- `--square 25`：方格真实边长 mm，按你打印的填。
**验收**：compute 打印 **重投影 RMS < 1px**；看 `undistorted/` 里图片边缘直线变直。
**做完发我**：RMS 数值。

---

## 6. 步骤 4 — E4 单应矩阵（相机/带面一动就重标）

**目的**：像素 → 传送带平面真实 mm。
**前提**：相机、传送带**已固定死**；E3 已完成。
**动作**：
1. 在传送带平面铺网格纸/直尺，让每个标定点的真实 mm 坐标可读。
2. 拍一张工作面静态图 `work_plane.png`（可用 `capture_pi.py`）。
3. 点 **6~9 个**坐标已知的点。

```bash
python homography.py pick   --image work_plane.png --out points.csv
python homography.py fit    --src points.csv --out outputs/homography.yaml
python homography.py verify --params outputs/homography.yaml --src points.csv
```

**原点怎么选**：选一个**好在画面里看见、又好用尺量到臂基座**的点（建议传送带靠臂这侧的角）。原点位置你自定，但输入的坐标必须是真实 mm。
**注意**：点要撒开覆盖整个工作区，别让 4 点共线或挤在中间。
**验收**：fit 打印 **mean 反投影误差 < 5mm**。
**做完发我**：mean / max 误差。

---

## 7. 步骤 5 — E5 臂基偏移（臂/带面一动就重标）

**目的**：把 E4 的"带面坐标系"钉到机械臂坐标系（一步平移）。
**动作**：卷尺量 E4 原点 → 臂基座 的偏移，手写 yaml：

```yaml
# outputs/arm_offset.yaml
offset_x_mm: <带面原点相对臂基座的 X 偏移>
offset_y_mm: <Y 偏移>
offset_z_mm: <带面相对臂基座的高度，通常负>
strawberry_grasp_z_mm: <草莓本身高度，例如 15>
```

**验收**：拿一个已知位置的草莓，跑 `coord_transform` 看算出的臂坐标对得上卷尺量的（差 < 8mm）。

## 8. 之后 — E6 / E7（集成测试，不算标定）

- **E6**：`python pi/main_pickup.py --dry-run` 走通 像素→臂坐标→关节角，不动臂，看路径不撞。
- **E7**：传送带摆 9 个网格点，跑实物抓取，统计成功率。
