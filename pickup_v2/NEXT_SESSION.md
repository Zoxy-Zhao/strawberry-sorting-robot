# pickup_v2 进度与技术复盘

> 本文件是 pickup_v2（任意位置草莓抓取）的进度总览与关键技术复盘。
> 想了解模块设计先读 [`docs/设计方案.md`](docs/设计方案.md) 与 [`docs/坐标系与IK.md`](docs/坐标系与IK.md)。

## 1. 它解决什么问题

把草莓抓取从 vision 主线的「固定姿态序列」升级为「视觉定位 + 逆运动学」：
相机识别草莓像素坐标 → 单应矩阵映射到工作面 mm → 加臂基偏移得到臂坐标 →
解析 IK 求 6 个关节角 → 串口 `K` 命令下发。整套实现与现有 vision 主线**完全隔离**，
删掉 `pickup_v2/` 目录原项目零痕迹。

## 3. 已完成的关键工作

**E1 关节标定**：base/wrist 确认为 1:1 线性映射；shoulder/elbow 因重力下垂呈非线性
（斜率 <1 且随角度变化），升级为**多点分段线性插值**（shoulder 4 点、elbow 4 点）。

**φ_fixed = −43° 固定腕部斜角**：垂直朝下（−90°）物理够不到传送带平面，改用固定斜角。
实测末端对地 + FK/IK roundtrip 双向验证一致，草莓抓取点 (301.5, −5.3, 97.4) mm 可达。

**代码质量**：`kinematics.ik()` 接入 φ_fixed 后 91 个单元测试全部通过；
经交叉审查修复 2 处缺陷（抓取流程误发已废弃的 `M` 命令、腕部旋转未标定致夹爪 roll 偏 80°）。

## 4. 核心技术发现：开环舵机的双关节重力下垂（本项目的主瓶颈）

这是 pickup_v2 最重要的工程结论，也是后续硬件选型的依据。

- **现象**：当前用开环 PWM 舵机（无角度编码器，无法读回真实关节角）。近点（r 小）
  肩关节抬高时**大臂垮**；远点（r 大）伸展时**小臂垮**；只有 r≈300mm（肩≈85°/肘≈85°）
  附近自然准（误差 ~3mm）。下垂量只与伸展半径 r 相关，与方位角无关。
- **长力臂放大效应**：小臂+夹爪共约 340mm，肘部 9° 的角度误差会让指尖位置偏 ~60mm。
  因此「自然准」的区间只有约 30mm 深（r 285~315），向外迅速恶化（r350 时偏 148mm）。
- **根因**：开环舵机无法闭环校正重力下垂。要在 100×100mm 全区达到一致精度，
  根本解是**改用总线舵机 / 带编码器舵机**（能读回真实角度做闭环），留作下一代机械臂方案。
- **当前缓解方案**：用「双关节下垂补偿」+ 关节角验证法（不依赖测量原点）——
  IK 给出目标角 → 实测实际角 → 下垂量 = 目标 − 实际 → 补偿命令 = 目标 + 下垂 → 复测，
  从中心 r300 向两端扩展直到舵机抬不动，由此标出真实可用的覆盖深度。

详细推导见 [`docs/标定演进复盘_2026-06-03.md`](docs/标定演进复盘_2026-06-03.md)。

## 5. 后续方向

1. 在开环硬件上用下垂补偿标出可用抓取区（预计深度 50~60mm）。
2. 完成 E3–E5 相机标定（像素 → 臂坐标）与触发逻辑。
3. 中长期：更换为总线/编码器舵机，实现真正的全区任意位置抓取。

## 6. 工作方式约定

物理标定遵循：先给理论期望值再实测、数据与物理矛盾时停下重测找根因、全面交叉验证；
量关节角（水平仪 + 角度尺）比量 3D 位置（卷尺）更可靠。
