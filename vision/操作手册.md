## 1. 环境准备

### 1.1 Pi 端安装依赖

```bash
# SSH 登录到树莓派
ssh pi@<你的Pi IP地址>

# 激活虚拟环境（之前已创建好的）
cd ~/vs_code
source .venv/bin/activate

# 安装依赖（3个包）
pip install opencv-python-headless ultralytics picamera2

# 验证安装成功
python -c "import cv2; print('OpenCV', cv2.__version__)"
python -c "from ultralytics import YOLO; print('Ultralytics OK')"
python -c "from picamera2 import Picamera2; print('Picamera2 OK')"
```

**预计耗时**：5-15 分钟（Pi 上 pip 比较慢）

**如果 picamera2 安装失败**：它通常是系统级预装的，试试：
```bash
# 用系统的 picamera2，不在虚拟环境装
pip install opencv-python-headless ultralytics
# 创建虚拟环境时加 --system-site-packages 以访问系统包
```

### 1.2 把代码拷贝到 Pi

**方法 A：SCP 远程拷贝**（在你的 Windows 电脑上执行）
```bash
# 把 vision/pi/ 下的所有 .py 文件拷到 Pi
scp <本地仓库路径>/vision/pi/*.py pi@<Pi_IP>:~/vs_code/strawberry_grasp/
```

**方法 B：U盘拷贝**
1. 把 `vision/pi/` 下的所有 `.py` 文件拷到 U 盘
2. U 盘插到 Pi
3. ```bash
   cp /media/pi/USB盘名/*.py ~/vs_code/strawberry_grasp/
   ```

**方法 C：直接在 Pi 上创建文件**
如果网络不方便，用 `nano` 逐个创建文件也行（不推荐，文件多）

### 1.3 确认 Pi 上的文件结构

```bash

```

## 3. 自采传送带数据

> **在哪运行**: Pi 上
> **目的**: 拍摄传送带场景下的草莓图片，补充训练数据
> **前提**: 摄像头测试通过（步骤 2）

### 3.1 准备工作

1. 准备真实草莓：
   - **成熟**：全红/深红的草莓，5-10 颗
   - **半成熟**：半红半绿/橙色的草莓，5-10 颗
   - **未熟**：绿色/白色的草莓，5-10 颗
2. 传送带通电，确保摄像头对准传送带工作区域
3. 如果有补光灯，打开

### 3.2 运行采集脚本

```bash
cd ~/vs_code/strawberry_grasp
source ../.venv/bin/activate

python capture_dataset.py
```

**启动后会看到**：
```
[Dataset] 保存目录: /home/pi/vs_code/strawberry_grasp/dataset_captured

正在启动 CSI 摄像头...
[Camera] CSI 摄像头已启动: 640x480

==================================================
草莓数据采集工具
==================================================
快捷键:
  r = 拍 ripe（成熟，红色）
  s = 拍 semi_ripe（半成熟，转色）
  u = 拍 unripe（未熟，绿色）
  c = 进入连拍模式（需先选类别）
  n = 查看统计
  q = 退出
==================================================
```

### 3.3 操作方法

**单拍模式**（推荐用于精确标注场景）：

1. 把一颗**红色成熟**草莓放在传送带上，摄像头对准
2. 输入 `r` 按 Enter → 拍一张，保存为 ripe 类别
3. 稍微移动草莓位置/角度，再按 `r` → 换个角度拍
4. 换一颗**半成熟**草莓，输入 `s` 按 Enter
5. 换一颗**未熟**草莓，输入 `u` 按 Enter
6. 输入 `n` 可以查看当前各类别已拍了多少张
7. 输入 `q` 退出

**连拍模式**（快速大量采集）：

1. 输入 `c` 按 Enter
2. 提示选择类别，输入 `r`（或 `s`/`u`）按 Enter
3. 程序每 0.5 秒自动拍一张，你只需要不断变换草莓的位置和角度
4. 按 Enter 停止连拍

### 3.4 采集建议

- **每类至少拍 50-80 张**（越多越好）
- **多变换条件**：
  - 不同位置（传送带左/中/右）
  - 不同角度（正面/侧面/倾斜）
  - 不同光照（开灯/关灯/自然光）
  - 单颗/多颗同时出现
  - 不同大小的草莓
- **拍完检查**：输入 `n` 看统计，确保三类数量大致均衡

### 3.5 采集完毕后

```bash
# 查看采集结果
ls dataset_captured/ripe/     | wc -l    # 成熟有多少张
ls dataset_captured/semi_ripe/ | wc -l   # 半成熟有多少张
ls dataset_captured/unripe/    | wc -l   # 未熟有多少张
```

**把数据拷贝回 Windows 电脑**（在 Windows 上执行）：
```bash
scp -r pi@<Pi_IP>:~/vs_code/strawberry_grasp/dataset_captured/ <本地仓库路径>/vision/dataset_captured/
```

### 3.6 标注自采数据

自采数据拍回来后**需要标注 bounding box** 才能训练。推荐用 **Roboflow**：

1. 打开 https://app.roboflow.com
2. 创建项目 → Object Detection → 类别填 `ripe, semi_ripe, unripe`
3. 上传 `dataset_captured/` 里的图片
4. 在线画框标注（拖拽画框，选类别）
5. 标注完成后导出 → 格式选 `YOLOv8` → 下载
6. 把下载的数据合并到本地数据集目录（如 `<数据集目录>\草莓dataset\`）里

**或者用 LabelImg（离线标注）**：
```bash
pip install labelImg
labelImg <本地仓库路径>/vision/dataset_captured/
```

## 4. PC 上训练模型

> **在哪运行**: 你的 Windows 电脑（有 GPU）
> **目的**: 用 YOLOv8n 训练草莓分类模型
> **前提**: 数据集准备好了（至少有 `<数据集目录>\草莓dataset` 的现有数据）

### 4.1 直接用现有数据训练（最快上手）

```bash
# 进入 vision 目录
cd <本地仓库路径>/vision

# 开始训练
python train.py
```

**训练过程会看到**：
```
[Detector] 正在加载模型: yolov8n.pt
Downloading yolov8n.pt...    ← 首次运行会下载预训练权重（~6MB）

Epoch 1/100:
  train/box_loss: 1.23  train/cls_loss: 2.45  train/dfl_loss: 1.56
  val/mAP50: 0.342  val/mAP50-95: 0.178

Epoch 2/100:
  ...

Epoch 100/100:
  val/mAP50: 0.891  val/mAP50-95: 0.623

训练完成！
最佳权重: runs/strawberry_v1/weights/best.pt
训练曲线: runs/strawberry_v1/results.png

正在测试集上评估...
测试集结果:
  mAP@0.5:     0.8856
  mAP@0.5:0.95: 0.6189
  ripe           AP@0.5=0.9123
  semi_ripe      AP@0.5=0.8534
  unripe         AP@0.5=0.8912
```

**预计耗时**：有 GPU 约 20-40 分钟，纯 CPU 约 2-4 小时

### 4.2 查看训练结果

训练完成后，在 `vision/runs/strawberry_v1/` 目录下会生成：

```
runs/strawberry_v1/
├── weights/
│   ├── best.pt          ← 最佳权重（你要用的）
│   └── last.pt          ← 最后一轮权重
├── results.png          ← 训练曲线图（loss、mAP 变化）
├── confusion_matrix.png ← 混淆矩阵（看哪些类容易混淆）
├── val_batch0_pred.jpg  ← 验证集预测可视化
└── args.yaml            ← 训练参数记录
```

**打开 `results.png` 查看**：
- 左侧 loss 曲线应该稳定下降
- 右侧 mAP 曲线应该稳定上升到 0.85+

**打开 `confusion_matrix.png` 查看**：
- 对角线数值高 = 分类正确率高
- 非对角线数值高 = 两类之间容易混淆

### 4.3 如果效果不好

**mAP < 0.85 的常见原因和解决方法**：

| 问题 | 症状 | 解决 |
|------|------|------|
| 数据太少 | loss 震荡不下降 | 补充数据，或增加 epoch 到 150-200 |
| 类别不平衡 | unripe 准确但 ripe 差 | 复制 ripe/semi_ripe 的图片让数量接近 |
| 过拟合 | train loss 很低但 val loss 高 | 减少 epoch，增强数据增强 |
| semi_ripe 和 ripe 混淆 | 混淆矩阵这两类交叉高 | 增加这两类的训练样本 |

### 4.4 合并自采数据后重新训练

如果你自采了传送带数据并标注好了：

1. 把标注好的图片和标签放入 `<数据集目录>\草莓dataset\images\train\` 和 `<数据集目录>\草莓dataset\labels\train\`
2. 重新运行 `python train.py`
3. 如果想从上次的权重继续训练（微调），修改 train.py：
   ```python
   model = YOLO("runs/strawberry_v1/weights/best.pt")  # 改这行
   ```
   并减少 epoch 到 30-50

## 6. 全链路联调

> **在哪运行**: Pi 上
> **目的**: 视觉检测 → 串口发指令 → MCU 执行抓取 → 完整闭环
> **前提**: 干跑测试通过（步骤 5），MCU 已上电并烧录状态机 v2

### 6.1 准备工作

1. **MCU 上电**，确认串口线连好（P109→Pi GPIO15, P110→Pi GPIO14, GND→GND）
2. **传送带通电**
3. 通过 DAP 串口（电脑连 MCU 的 USB）进入自动模式：
   - 打开串口工具（如 PuTTY），连接 MCU 的 DAP 串口
   - 发送 `a` → 开启自动模式
   - 发送 `g` → 启动运行（MCU 进入 BELT_RUN 状态，传送带开始转）

### 6.2 启动视觉系统

```bash
# SSH 到 Pi
cd ~/vs_code/strawberry_grasp
source ../.venv/bin/activate

# 正式运行（连接 MCU）
python main.py
```

**正常输出**：
```
========================================
草莓分拣视觉系统启动
模式: 正常(连接MCU)
========================================
正在初始化摄像头 (csi)...
[Camera] CSI 摄像头已启动: 640x480
正在加载 YOLO 模型...
[Detector] 模型加载完成，置信度阈值=0.5
正在连接 MCU 串口...
串口已打开: /dev/serial0 @ 115200
Ping 成功，MCU 回复: PONG
MCU 连接成功
主循环开始，Ctrl+C 退出
--------------------------------------------------
```

### 6.3 操作流程

1. **放草莓**：把一颗草莓放在传送带起始端
2. **传送带运输**：草莓随传送带经过摄像头视野
3. **自动检测**：系统识别到草莓，打印分类结果
4. **自动发指令**：串口发送 A/B/C 给 MCU
5. **MCU 执行**：机械臂抓取并放到对应位置
6. **冷却等待**：等 12 秒（MCU 完成动作），然后继续检测下一个
7. 按 `Ctrl+C` 退出

### 6.4 调整参数

如果联调过程中需要调整，修改 Pi 上的 `config.py`：

```python
# 如果误检太多（把背景当草莓），调高置信度
CONFIDENCE_THRESHOLD = 0.7   # 从 0.5 调到 0.7

# 如果漏检（草莓经过但没检测到），调低置信度
CONFIDENCE_THRESHOLD = 0.3   # 从 0.5 调到 0.3

# 如果 MCU 动作还没做完就发了下一个指令，增加冷却时间
DETECT_COOLDOWN = 15.0       # 从 12 调到 15

# 如果只想检测画面中央区域（避免检测到边缘的东西）
DETECT_REGION = (0.2, 0.2, 0.8, 0.8)
```

修改后不需要重新训练模型，直接重启 `python main.py` 即可。

### 6.5 紧急停止

如果机械臂出现异常：
- **Pi 端**：按 `Ctrl+C` 停止视觉系统
- **MCU 端**：通过 DAP 串口发送 `x` 紧急停止

## 7. 常见问题排查

### 摄像头相关

| 问题 | 解决 |
|------|------|
| `Camera not detected` | 关机 → 重新插 CSI 排线 → 开机 |
| `libcamera` 相关错误 | `sudo apt update && sudo apt install python3-libcamera libcamera-apps` |
| 画面很暗 | 检查补光灯；或在 camera.py 的 cam_config 里设置曝光参数 |
| 画面模糊 | CSI 摄像头需要手动对焦（旋转镜头） |

### 模型相关

| 问题 | 解决 |
|------|------|
| `FileNotFoundError: models/strawberry_yolov8n.pt` | 权重文件没拷贝过来，重做步骤 5.1 |
| FPS 太低 (<5) | config.py 里降低 `CAMERA_WIDTH=320, CAMERA_HEIGHT=240` |
| 一直检测不到草莓 | 降低 `CONFIDENCE_THRESHOLD = 0.3`，或者确认模型权重正确 |
| 分类经常错 | 需要补充传送带场景的自采数据重新训练 |

### 串口相关

| 问题 | 解决 |
|------|------|
| `Permission denied: /dev/serial0` | `sudo chmod 666 /dev/serial0` 或 `sudo usermod -aG dialout pi` |
| `Ping 超时，MCU 无回复` | 检查接线（TX→RX 交叉连）、MCU 是否上电、波特率是否 115200 |
| MCU 收到指令但不动 | 确认 MCU 处于自动模式（发过 `a` 和 `g`） |

### 训练相关

| 问题 | 解决 |
|------|------|
| `CUDA out of memory` | train.py 里 `batch=16` 改小为 `batch=8` |
| 训练很慢（纯 CPU） | 确认安装了 CUDA 版本的 PyTorch：`pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121` |
| mAP 一直上不去 | 增加 epoch、补充数据、检查标注质量 |
